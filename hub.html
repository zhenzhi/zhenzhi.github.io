Enterprise Search Management

 

Company

Voyager Search is a leading global provider of geospatial, enterprise search solutions that enable a single point of search across all of your content repositories. With Voyager, users can search, find, and deliver more than 2,000 types of content. Built on top of Solr/Lucene, the world’s premier open source solution, Voyager offers Fortune 500 businesses, government entities, and multinational organizations easy-to-use search, data management, document delivery, and map visualization capabilities. We do search technologies.

 

As the first and the only designer in the team, I led the end to end design from the research to the visual. I collaborated with product owner and developers to define the product need and push the pixels.

Product

Voyager HQ is a central control panel application for managing all the searching works including data integrating, indexing, processing, and security work. You can consider this as an administrative tool to control all the search works.

 

Since I joined Voyager Search, my main task is redesigning the whole Voyager HQ. The redesigned new product is part of new Voyager suite - Voyager Vose, and already released in 2019 Q1. See website introduction.

Why do we need a redesign

Technological Need

Since the initial Voyager product was launched our customers have experienced massive growth in the amount of structured and unstructured data they need to index with a search and discovery platform such as Voyager. A better approach to manage the process of building a single large index without the administrator having to configure federation or to run merge tasks after multiple indexes have been created. And also in the old UI, we used Apache Wicket for our front-end work, which is hard to scale compared to other frameworks like AngularJS and Reacts. That's the reason we need a new design to match that need. With new Voyager HQ, the scale out of the discovery tier/nodes will be handled.

Design Need

The old version of Voyager HQ was designed by developers but doesn’t make sense for them either. We have a lot of good features but never thought how to designed that to make stuff easier to understand and easier to use. The downside of that is we received a lot of help tickets from our customers. They don’t know how to use a specific feature or they can’t find it. This causes a huge burden on customer team because they have to answer a lot of emails and sometimes have to schedule a video call to explain that. Bad designs also build a bad reputation and impression of the whole platform, which damages business. The old UI is not looking good, users feel they are using a very old and never updated software. Our goal also includes giving users a pleasant feeling when using our product. A new visual appealing UI seems necessary.

What's the Goal

Goals

  • Design a new UI to suit new technology structure
  • Provide users with better user experience
  • Decrease amount of helping tickets
  • Release first version in 10 months

What is this

Search data need to be cleaned and processed then can be used. Pipeline provides functions to transform and manipulate the properties (metadata) of data records as it adds them to the Index. For example, location data like “Austin, TX” need to process through geotagging pipeline then it can be searched in a map.
